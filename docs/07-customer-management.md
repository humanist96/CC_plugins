# 튜토리얼 07: 고객 관리와 지원

> 난이도: 중급 | 소요시간: 20분

## 학습 목표

- Customer Support 플러그인으로 고객 문의 처리 자동화하기
- Enterprise Search로 사내 정보 통합 검색하기
- 고객 응대부터 KB 문서 작성까지 전체 워크플로우 실습

## Customer Support 명령어 총정리

| 명령어 | 기능 |
|--------|------|
| `/customer-support:triage` | 티켓 분류 및 우선순위 지정 |
| `/customer-support:draft-response` | 고객 응답 초안 작성 |
| `/customer-support:research` | 고객 문의 조사 |
| `/customer-support:escalate` | 이슈 에스컬레이션 |
| `/customer-support:kb-article` | 지식 베이스 문서 작성 |

## Enterprise Search 명령어

| 명령어 | 기능 |
|--------|------|
| `/enterprise-search:search` | 통합 검색 |
| `/enterprise-search:digest` | 주간/일간 다이제스트 |

## 실습 1: 티켓 분류 (Triage)

### 시나리오
새로운 고객 문의가 들어왔습니다. 우선순위를 판단하고 적절한 담당자에게 배정합니다.

### 1-1. 긴급 이슈 분류

```
/customer-support:triage
```

입력:

```
다음 고객 문의를 분류해줘:

고객: KB국민은행
담당자: 김부장
문의 내용:
"오늘 아침부터 실시간 시세 데이터가 10분 이상 지연되고 있습니다.
트레이딩 룸에서 업무가 불가한 상황이라 긴급히 처리 부탁드립니다.
영향 범위: 트레이딩 데스크 전체 (약 50명)"

서비스: 실시간 시세 데이터 피드
SLA: 99.9% 업타임, 장애 복구 30분 이내
계약 등급: Premium
```

결과에 포함되는 항목:
- 우선순위 (P1~P4)
- 카테고리 분류
- 담당팀 라우팅
- 예상 해결 시간
- 초기 대응 가이드

### 1-2. 일반 문의 분류

```
다음 3건의 문의를 한번에 분류하고 우선순위를 매겨줘:

문의 1: "API 연동 가이드 문서가 어디 있나요?" - 신한은행 개발팀
문의 2: "월별 사용량 리포트를 자동으로 받을 수 있나요?" - 하나금융 운영팀
문의 3: "계약 갱신 시 추가 할인이 가능한지 문의드립니다" - NH투자증권 구매팀
```

## 실습 2: 고객 응답 작성

### 2-1. 장애 대응 응답

```
/customer-support:draft-response
```

입력:

```
다음 상황에 대한 고객 응답을 작성해줘:

상황: 실시간 시세 데이터 10분 지연 장애
고객: KB국민은행 김부장
우선순위: P1 (Critical)
현재 상태: 엔지니어링팀 조사 중, 원인 파악 진행 중
예상 복구: 1시간 이내

톤: 공식적이면서도 진정성 있게
포함 사항:
- 사과 및 현황 공유
- 복구 예상 시간
- 임시 대안 안내
- 후속 조치 계획
```

### 2-2. 기능 요청 응답

```
다음 고객 요청에 대한 응답을 작성해줘:

고객: 하나금융 운영팀
요청: "월별 사용량 리포트를 매월 1일에 자동으로 이메일로 받고 싶습니다"
현재 상태: 해당 기능은 다음 분기 로드맵에 있음
대안: 현재 대시보드에서 수동 다운로드 가능

톤: 친절하고 건설적
```

### 2-3. 다국어 응답

```
다음 영문 문의에 대해 영어와 한국어 두 가지 버전으로 응답을 작성해줘:

문의: "We're experiencing intermittent connectivity issues with the WebSocket API since yesterday. Could you please investigate?"

고객: Global Investment Bank (해외 고객)
```

## 실습 3: 고객 문의 조사

### 3-1. 기술 문의 조사

```
/customer-support:research
```

입력:

```
다음 기술 문의를 조사하고 답변을 준비해줘:

문의: "REST API에서 대량 주문 조회 시 타임아웃이 발생합니다.
한 번에 10,000건 이상 조회하면 30초 제한에 걸립니다."

조사 항목:
- API 페이지네이션 지원 여부
- 대량 조회 최적화 방법
- 타임아웃 설정 변경 가능 여부
- 벌크 다운로드 API 존재 여부
```

### 3-2. 계정/계약 관련 조사

```
다음 고객의 계약 현황을 정리해줘:

고객: NH투자증권
현재 계약: Premium 플랜 (2024.01 ~ 2025.12)
월 이용료: 3,000만원
사용 서비스: 실시간 시세, 히스토리컬 데이터, API
문의: 계약 갱신 시 Enterprise 플랜 업그레이드 조건

Enterprise 플랜 기준:
- 월 5,000만원
- 전용 서버, SLA 99.99%, 전담 지원
- 추가: 뉴스 피드, 대체 데이터

업그레이드 시 혜택과 비용 차이를 분석해줘
```

## 실습 4: 에스컬레이션

### 4-1. 기술 이슈 에스컬레이션

```
/customer-support:escalate
```

입력:

```
다음 이슈를 엔지니어링팀에 에스컬레이션해줘:

이슈: WebSocket 연결 간헐적 끊김
고객: KB국민은행 (Premium 고객)
영향: 트레이딩 데스크 50명, 약 3시간 지속
빈도: 최근 일주일간 매일 오전 9시~10시 사이 발생

이미 시도한 조치:
- 클라이언트 측 재연결 로직 확인 완료
- 네트워크 레이턴시 정상 (< 5ms)
- 로드밸런서 로그에서 5xx 에러 발견

비즈니스 임팩트:
- 계약 갱신 12월 예정 (연 3.6억 계약)
- 고객 불만 3회 접수
- 경쟁사 전환 위험

에스컬레이션 대상: 백엔드 엔지니어링팀
긴급도: High
```

## 실습 5: 지식 베이스 문서 작성

### 5-1. FAQ 문서 작성

```
/customer-support:kb-article
```

입력:

```
다음 자주 묻는 질문에 대한 KB 문서를 작성해줘:

주제: API 연동 시 인증 방법
대상: 고객사 개발팀

포함 내용:
1. API Key 발급 방법
2. OAuth 2.0 인증 플로우
3. 토큰 갱신 방법
4. 인증 오류 코드 및 해결법
   - 401 Unauthorized
   - 403 Forbidden
   - 429 Rate Limited
5. 보안 모범 사례
```

### 5-2. 트러블슈팅 가이드

```
다음 해결된 이슈를 기반으로 트러블슈팅 가이드를 작성해줘:

이슈: 대량 데이터 조회 시 타임아웃
해결: 페이지네이션 적용 + cursor 기반 조회

단계:
1. 문제 증상 설명
2. 원인 분석
3. 해결 방법 (코드 예시 포함)
4. 예방 조치
```

## 실습 6: 통합 검색

### 6-1. 사내 정보 검색

```
/enterprise-search:search
```

입력:

```
"KB국민은행 WebSocket 장애" 관련 내부 문서, 이전 티켓, 기술 문서를 모두 검색해줘
```

### 6-2. 주간 다이제스트

```
/enterprise-search:digest
```

입력:

```
이번 주 고객 지원 활동 다이제스트를 생성해줘:
- 접수된 티켓 현황
- 해결된 이슈 요약
- 미해결 이슈 목록
- 주요 고객 피드백
```

## 전체 워크플로우 실습

### 고객 문의 처리 End-to-End

다음 시나리오를 순서대로 실행해보세요:

```
1단계: 티켓 접수 & 분류
/customer-support:triage
→ "KB국민은행에서 API 응답 지연 문의가 왔습니다..."

2단계: 조사
/customer-support:research
→ "API 응답 지연 원인을 조사해줘..."

3단계: 고객 응답
/customer-support:draft-response
→ "조사 결과를 바탕으로 고객 응답을 작성해줘..."

4단계: (필요시) 에스컬레이션
/customer-support:escalate
→ "엔지니어링팀에 에스컬레이션..."

5단계: KB 문서화
/customer-support:kb-article
→ "해결된 이슈를 KB 문서로 작성해줘..."
```

## 핵심 팁

1. **고객 등급 명시**: Premium, Standard 등 고객 등급에 따라 응대 톤과 우선순위가 달라집니다
2. **SLA 확인**: 에스컬레이션 시 SLA 조건을 반드시 포함하세요
3. **이력 참조**: 이전 티켓이나 대화 이력을 제공하면 더 맥락 있는 답변을 받습니다
4. **KB 문서화**: 반복 문의는 즉시 KB 문서로 작성하면 향후 효율이 높아집니다
